# Security Policy

## Supported versions

LoomPress currently supports the latest code on `main`.

## Reporting a vulnerability

Do not open public issues for security reports.

When this repository is public, enable GitHub private vulnerability reporting and use that channel for all disclosures. Until that is enabled, contact the repository maintainer privately and avoid sharing exploit details in issues, discussions, or pull requests.

## Disclosure expectations

- Give the maintainer a reasonable amount of time to investigate and patch the issue.
- Include reproduction steps, affected files or routes, and impact.
- If credentials or tokens are involved, rotate them immediately after reporting.
